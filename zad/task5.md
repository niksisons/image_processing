# 4. Пояснительная записка

## Обоснование структуры календарного плана

 Работа сгруппирована по этапам — аналитика, проектирование, реализация, тестирование, оформление. Такая структура соответствует методологии итеративной разработки: каждую стадию можно завершить независимо, но каждая следующая зависит от предыдущей.

## Методология анализа рисков

Для анализа применена матрица риска: вероятность × влияние. Выбраны риски, которые характерны именно для локальных DevOps-инфраструктур: конфигурации Kubernetes, сбои GitLab Runner, ошибки кастомных операторов, нехватка производительности, срывы сроков. Для каждого риска определена стратегия: предотвращение, смягчение или принятие.

## Принципы выбора контрольных точек

Контрольные точки выбраны по принципу «главных технических вех»: архитектура, CI/CD, интеграционные тесты, production, оформление диплома. Это позволяет отслеживать прогресс и вовремя корректировать график. Все точки строго завязаны на календарный план и ключевые технические события проекта.

## Взаимосвязь всех компонентов системы

 В итоге получается единая система управления проектом: план → риски → контроль → корректировка. Система гарантирует, что проект будет завершён вовремя.
