audit_id — имя атрибута (поля таблицы);

<<PK>> — Primary Key (первичный ключ), то есть уникальный идентификатор записи.

<<FK>> — Foreign Key (ключ зависимости)

Каждый коммит (commit) принадлежит одному репозиторию (repository).

Из коммита запускается пайплайн (pipeline).

Пайплайн состоит из задач (ci_job), каждая порождает артефакты (artifact).

Пайплайн создаёт Docker-образ (image), который хранится в реестре (registry).

В окружение (environment) разворачивается релиз (release), который использует конкретный образ.

Микросервисы (service) генерируют метрики (metric), которые могут вызвать алерты (alert).

Все действия пользователя фиксируются в audit_log.
